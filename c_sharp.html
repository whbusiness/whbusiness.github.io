Another group project I worked on was a game called ScrollRoll made by 2 programmers (including me), 2 game artists and 2 game designers. I had to make sure my code worked with others in this project especially. The two main parts of the game I worked on was the AI and the camera. In this game I decided to use rigidbody AIs rather than navmesh agents due to the game being a physics based game where you try to push and pull the other players (or AI) off the map. The game is local multiplayer and I worked on including AI to fill the game up to 4 people (AI would replace each missing person). The video below shows 1 player (purple) and the rest are AI. The AI works by picking a random target out of all the players (itself excluded)It then checks to see if it is within range, if it is then it will push them (the push has a cooldown). The AI will look in the direction of the target and will always move forward using rigidbody.addforce
